1. Introduction

Plexus is a software program that performs Gibbs sampling for the purposes of Approximate Bayesian estimation of posterior distributions of nodes in a Bayesian network. It is coded in Swift for macOS, with a computation kernel written in OpenCL to allow parallel processing of the Gibbs sampler.

2. Data Model

Data in Plexus is organized into Entries, each of which can contain any number of Traits and each of may belong to any number of Structures.

2.1. Entries

An Entry is the basic container for data in Plexus. An Entry in Plexus is the basic container for data, and may represent any type of individual notional unit, from the contents of an archaeological dig square to an aspect of a culture to , (see Use Cases below). Entries are arranged in a hierarchical tree structure.

3. Importing Data

Plexus accepts CSV formatted data files. It expects the same number of columns in each row as there are columns in the header (first) line. the header line names the traits. The column names "Name" and "Structure" are reserved, and if deteceted in the header line become the name of the Entry and/or its Structure. Each row is a spearate Entry. Within one comma-delimited entry, a tab (\t) will be read as two separate insatnces of a single trait (this is useful for importing exports from a database such as Filemaker).

NOTE: Filemaker currently exports multiple entries in between two commas with an \r separator, not a \t. This can cause lots of problems if, for example, you need to open your input file in a text editor before importing it into Plexus, since the \r's will be read as newlines. To convert for use in Plexus, simply open your Filemaker export in a text editor, and use find-replace to replace all \r's with \t's, and also make sure to use Unix-style (LF) line endings.
